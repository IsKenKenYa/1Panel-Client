# 日志系统优化与隐私保护计划 (修订版)

## 目标 (Summary)
优化客户端日志系统，主要包括：
1. **公网 IP 自动脱敏**：在日志落盘和输出前，自动识别并隐藏公网 IP 地址。保留内网 IP 以便调试。
2. **风险与测试建议告知**：在用户导出日志和反馈中心页面增加提示：推荐使用内网虚拟机进行测试（当前为预览版）；提醒用户检查敏感信息且不要随意发送日志。
3. **日志清晰度优化**：减少非错误日志中不必要的堆栈打印，优化日志输出格式，使其更易读。

## 现状分析 (Current State Analysis)
1. `LoggerConfig.maxMethodCount` 当前为 3，导致普通的 `info` 或 `debug` 日志都会输出 3 层函数调用栈，干扰阅读。
2. `LoggerConfig.enableFileOutput` 在 release 模式下为 `!kReleaseMode`，意味着 Release 包默认不写本地文件，这会导致用户在正式版中无法导出有效的业务日志。
3. 日志目前直接输出，IP 等敏感信息均以明文形式保存在本地和日志文件中。
4. 反馈中心尚未包含预览版的测试建议及隐私免责说明。

## 提议的更改 (Proposed Changes)

### 1. 公网 IP 自动脱敏
- **修改文件**：`lib/core/services/logger/logger_service.dart`
- **方案**：在 `_AppLogOutput` 中加入 IP 正则匹配及内网 IP 判定逻辑。匹配到 IP 地址后，判断是否属于内网网段（如 `10.x.x.x`, `172.16.x.x-172.31.x.x`, `192.168.x.x`, `127.x.x.x` 等）。若是内网 IP 则原样保留，若是公网 IP 则替换为 `***.***.***.***`。这保证了测试阶段内网信息可见，而用户真实公网 IP 不会被导出。

### 2. 日志清晰度与配置优化
- **修改文件**：`lib/core/config/logger_config.dart`
- **方案**：
  - 将 `maxMethodCount` 从 3 改为 0（普通日志不打印堆栈，只保留信息），`maxErrorMethodCount` 保持 8（错误发生时保留堆栈以便排查）。
  - 将 `enableFileOutput` 更改为 `true`，确保所有环境下均可产生日志文件供用户导出反馈。

### 3. 页面提示与导出风险告知
- **修改文件**：`lib/features/settings/feedback_center_page.dart`
- **方案**：
  - 修改反馈中心的介绍文本，或者在导出前加入提示。在 `_exportAppLogs` 触发时弹出 `AlertDialog` 进行确认。
  - 弹窗/页面文案将结合用户的要求：“当前为预览体验发布，推荐使用内网 IP（如内网 1Panel 虚拟机）进行测试，以避免潜在影响。日志中可能包含部分敏感信息，系统已自动对公网 IP 进行脱敏处理。请勿将日志随意发送给无关人员，并在反馈前自行检查是否遗留了其他敏感信息。”
- **修改文件**：`lib/l10n/app_zh.arb` 和 `lib/l10n/app_en.arb`
- **方案**：新增以下国际化词条：
  - `settingsFeedbackLogExportWarningTitle`: 导出日志与测试提示
  - `settingsFeedbackLogExportWarningMessage`: 当前为预览体验发布，推荐使用内网 IP（内网 1Panel 服务器或虚拟机）进行测试以避免影响。\n\n系统已自动对日志中的公网 IP 等信息进行脱敏处理。请勿将日志随意发送给无关人员，并在反馈问题前自行检查是否还有其他敏感信息遗留。
  - `settingsFeedbackLogExportConfirm`: 确认导出

## 假设与决策 (Assumptions & Decisions)
- **判定内网 IP**：通过正则表达式提取出 IPv4 后，用代码逻辑判断第一、第二段数字来确定是否为内网 IP，比写死在正则中更具可读性和准确性。
- **页面提示**：将内网测试推荐和隐私警告合并放在导出日志时的弹窗中，或者放在反馈中心页面顶部，以确保用户在分享/导出前一定会看到该建议。我们将选择放在“导出日志弹窗”中作为强提醒。
- **取消默认堆栈**：去掉常规日志的 `methodCount` 能大幅提升日志信噪比。

## 验证步骤 (Verification Steps)
1. 运行应用，使用 `appLogger.i` 打印公网 IP（如 `8.8.8.8`）和内网 IP（如 `192.168.1.100`）。
2. 验证控制台和文件中，公网 IP 变为 `***.***.***.***`，而内网 IP 保持 `192.168.1.100`。
3. 触发一次普通的 `info` 日志，确认没有多余的调用堆栈，日志变得干净。
4. 进入“设置 -> 反馈中心”，点击“导出日志”。
5. 确认弹出包含内网测试推荐和敏感信息警告的对话框，点击“确认导出”后正常保存日志文件。
