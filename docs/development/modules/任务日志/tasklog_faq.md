# 任务日志模块 FAQ

## 1. 现在去哪里查看任务日志？

入口在日志中心：

- `LogsCenterPage` -> `Task` tab

点击列表项后进入 `TaskLogDetailPage`。

## 2. 任务日志详情的数据来自哪里？

- 列表元信息来自 `POST /logs/tasks/search`
- 正文日志来自 `POST /files/read`

所以详情页并不是请求一个单独的 `task detail` REST 接口。

## 3. 为什么不把 task log 做成单独导航模块？

因为上游语义更接近“日志中心的一部分”，而不是独立产品域。移动端把它放在日志中心里，可以减少信息架构重复。

## 4. 当前支持哪些筛选？

- 状态筛选
- 类型筛选

这满足 Week 6 的 MVP 范围，后续再考虑时间范围等高级筛选。

## 5. 任务执行中的日志会怎么显示？

- 列表顶部会显示 executing count
- 详情页支持 refresh
- 正文查看器会保持文本浏览和复制能力

## 6. no-server 时会发生什么？

不会自动请求任务日志 API；页面会保持 server-aware 的空态行为。
